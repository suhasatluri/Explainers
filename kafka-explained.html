Why It's Fast

Why Kafka is stupidly fast

Drag the lever to add more printing presses. Watch how many papers per second the depot can push out.

Printing presses (partitions) ×1
Papers per second 90k

It prints in order

Kafka only ever adds to the end of a stack — it never stops to slot a page into the middle. Writing in a straight line is the fastest thing a computer can do, like a press running non-stop.

Many presses at once

A newspaper can be split across several presses (partitions) running side by side. Twice the presses, roughly twice the output. Need more? Add presses.

It ships in bundles

Instead of sending one paper per trip, Kafka loads a whole truck and sends them together. Fewer trips, far more papers delivered.

+ For the curious: the "no photocopy" trick

When a reader collects their papers, the depot hands them straight from the shelf onto the delivery truck — without making a fresh photocopy in the back office first. Engineers call this zero-copy: data skips an extra trip through the computer's memory and goes almost directly to the network, saving huge amounts of effort.

Keeping Order

With so many presses, how does order survive?

This is the one thing people get wrong. Kafka reads each story's key — here, which customer it belongs to — and always sends that key to the same press. So one customer's events can never overtake each other. File a few and watch where they land.

Press 2

Same key → same press, every time. Inside a press the order is guaranteed (#1, #2, #3…). Across different presses there's no global order — and that's fine, because related stories always share a press. Old stories roll off into the archive once the shelf is full.

The Whole Newsroom

Kafka isn't just a delivery truck

It's tempting to think Kafka only carries data from A to B. In reality it's a whole newsroom: it can pull stories in automatically, rewrite and combine them as they arrive, and ship the results out — without anyone hand-writing plumbing code. Tap a stage to follow a story through.

The Last Word

If you remember one thing: Kafka is a newspaper for data — writers file events, the depot keeps them in order on the shelf, and any number of readers subscribe and catch up at their own pace.

  • It's a central hub, so systems stop wiring directly to each other
  • Messages are kept (an archive), not deleted when read
  • It splits work across partitions to scale — while keeping each key in order
  • Many readers can read the same stream, independently
  • It's a platform: it can connect and process data, not just move it
